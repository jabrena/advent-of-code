# Problem Type Classification - Advent of Code 2015

This document classifies Advent of Code 2015 problems by their primary problem type.

## Classification Categories

- **Data Processing**: Problems involving parsing, transforming, filtering, or aggregating data
- **Grid (2D Matrix)**: Problems involving 2D grid/matrix manipulation, pathfinding, or spatial operations
- **Games & Algorithms**: Problems involving game simulation, game rules, or algorithmic challenges like recursion, backtracking, dynamic programming
- **Others**: Problems that don't fit into the above categories

## Detailed Analysis

### Day1 - Lisp
**Type**: Data Processing

**Description**: 
- Counts parentheses to determine final floor
- Part 2 finds first position where floor becomes -1

**Key Operations**:
- Character-by-character processing
- State tracking (floor level)
- Position tracking

### Day2 - Present
**Type**: Data Processing

**Description**:
- Parses present dimensions (L×W×H)
- Calculates wrapping paper area and ribbon length

**Key Operations**:
- Regex parsing
- Surface area calculation
- Volume and perimeter calculation

### Day3 - Houses
**Type**: Data Processing

**Description**:
- Tracks Santa's position based on directions
- Counts unique visited houses
- Part 2 alternates between Santa and Robo-Santa

**Key Operations**:
- Position tracking (2D coordinates)
- Set-based visited tracking
- Direction parsing

**Note**: Uses 2D coordinates but not a grid structure - just position tracking

### Day4 - AdventCoinMiner
**Type**: Data Processing

**Description**:
- Finds MD5 hash starting with specific prefix
- Brute force search through numbers

**Key Operations**:
- MD5 hash generation
- Prefix matching
- Sequential search

### Day5 - StringValidator
**Type**: Data Processing

**Description**:
- Validates strings based on multiple rules
- Part 2 has different validation rules

**Key Operations**:
- String pattern matching
- Regex validation
- Character sequence checking

### Day6 - LightCounter
**Type**: Grid (2D Matrix)

**Description**:
- Manipulates 1000×1000 grid of lights
- Processes turn on/off/toggle commands on rectangles
- Part 2 tracks brightness instead of on/off

**Key Operations**:
- 2D grid manipulation
- Rectangle operations
- Brightness tracking

### Day7 - Circuit
**Type**: Data Processing

**Description**:
- Processes wire circuit instructions
- Resolves dependencies and propagates values
- Part 2 overrides wire 'b' with Part 1 result

**Key Operations**:
- Dependency resolution
- Bitwise operations
- Value propagation

### Day8 - StringLiteralCalculator
**Type**: Data Processing

**Description**:
- Calculates difference between string literal and in-memory representation
- Part 2 calculates encoded length

**Key Operations**:
- String parsing
- Character escaping handling
- Length calculation

### Day9 - RouteOptimizer
**Type**: Games & Algorithms

**Description**:
- Traveling Salesman Problem variant
- Finds shortest/longest route visiting all cities
- Generates all permutations

**Key Operations**:
- Permutation generation
- Distance calculation
- Optimization (min/max finding)

### Day10 - LookAndSay
**Type**: Data Processing

**Description**:
- Generates Look-and-Say sequence
- Applies sequence transformation multiple times

**Key Operations**:
- Run-length encoding

### Day11 - PasswordValidator

**Description**:

- String replacement

**Description**:

- Register manipulation
- Program counter management

### Day24 - Package

**Key Operations**:
- Combination generation
- Subset sum problem
- Optimization (min quantum entanglement)

### Day25 - Code
**Type**: Data Processing
